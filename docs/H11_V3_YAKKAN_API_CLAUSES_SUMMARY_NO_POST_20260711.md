# 外国為替FX取引約款 — API関連条項サマリー（no-POST・safe summary）

Date: 2026-07-11

Source: [外国為替FX取引約款PDF](https://coin.z.com/corp_imgs/policy/kawasefx-yakkan.pdf?ver=20250208)
（2025-02-08改定版）

著作権の関係で原文の長文引用は避け、条項番号と要旨のみ記す。法的判断が必要な場合は
原文および専門家確認をoperatorが行うこと。本書はH-11 v3 activation review用の
参考情報であり、法的助言ではない。

## 結論

**個人の自動売買（API経由の注文）を禁止する条項は存在しない。** 第8条2項が取引画面操作と
外国為替FX APIを対等な公式注文経路として明記しており、API利用そのものは想定内の利用形態。

## 関連条項

| 条項 | 要旨 |
|---|---|
| 第3条（自己責任の原則） | 本取引はお客様自身の判断と責任において行う |
| 第8条2項 | 注文受付は「取引画面操作」または「外国為替FX API」の2経路のみ。API注文は正式な注文手段 |
| 第9条(4) | 注文時の入力事項に指値・逆指値・OCO・IFD・**IFD-OCO**注文を明記。IFDOCOは約款上も正式な注文種別 |
| 第11条 | 証拠金不足・約款違反・「当社が不適切と認める場合」は注文を拒否/不執行。(3)号はGMO側の裁量が広い点に留意 |
| 第13条（取引の取消し） | 指定外方法での注文・市場実勢を反映しない約定価格・明白な誤り/不正手段による取引は、確認後でもGMOが取消可能。GMOは無責任 |
| 第17条（ロスカット） | 証拠金維持率が基準を下回ると自動ロスカット。全未約定注文の失効＋全建玉の強制決済。証拠金維持に留意 |
| **第20条（APIの利用許諾）** | 譲渡不可・再許諾不可の非独占的利用権。**GMOは通知だけで許諾内容の変更・取消が可能、かつ無責任** |
| **第21条3項（禁止行為）** | 商業利用・第三者への販売/利用許諾・解析（リバースエンジニアリング）・改竄・機能変更・再配信を禁止。**個人の自動売買自体は対象外** |
| **第22条1項（credential管理義務）** | APIキー/シークレットを第三者に開示・貸与・共有せず、漏洩しないよう厳重管理する義務 |
| **第22条2項（credential利用のみなし規定）** | operatorのcredentialでの利用は全てoperator自身の利用とみなす。第三者不正利用時もoperatorが責務を負う（例外: GMOの故意・重過失、またはoperatorに帰責事由なし） |
| **第22条3項（システム誤動作の免責）** | API管理不備・使用上の過誤・第三者使用による損害はGMO無責任（例外: GMOの故意・重過失による債務不履行/不法行為のみ） |

## H-11 v3設計への示唆（アクション不要・記録のみ）

- 第20条によりAPI利用許諾がいつでも通知だけで取消され得る＝接続断・認証失敗は構造的にあり得る事象。
  既存の「unknown/failure時はHALTしretry/repostしない」設計で対応済み、追加対応不要。
- 第22条3項により、H-11 v3自体のバグ・誤発注・二重発注等による損失はoperator全責任。
  これはACTIVE policyの「実験の価格を自分で払う」という前提と整合しており、
  凍結予算・停止基準・one-attempt制約の必要性を裏付ける。
- 第17条のロスカット時「全未約定注文の失効」は、v3のIFDOCO pending entryにも適用され得る。
  証拠金に十分な余裕を持たせる運用（固定10,000通貨・budget上限想定なら通常問題にならない）。
- 第13条の事後取消は極端な価格異常時のみの想定で、v3の通常運用への影響は限定的。

```text
prohibited_automated_trading_clause_found=false
api_ordering_officially_recognized=true
customer_bears_own_system_bug_liability=true
gmo_can_revoke_api_license_by_notice_no_liability=true
gmo_can_retroactively_cancel_abnormal_trades=true
```
